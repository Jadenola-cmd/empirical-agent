## 1. 平台概述

平台分为**两层**，按顺序操作：

```
第一层：数据清洗
  上传文件 → 字段映射（可选）→ 合并配置（多文件）→ 清洗配置 → 执行清洗

第二层：实证分析
  选择分析方法 → 配置变量 → 开始分析 → 查看结果 → 导出
```

清洗完成后，结果自动传入第二层，无需重新上传。

---

## 2. 支持的文件格式

| 格式 | 说明 |
|------|------|
| `.csv` | 逗号分隔文本，UTF-8 编码 |
| `.xlsx` | Excel 工作簿（2007 及以上） |
| `.dta` | Stata 数据文件 |

**不支持** `.xls`（旧版 Excel）。最多同时上传 **5 个文件**。

## 3. 第一层：数据清洗

### 3.1 上传文件

点击上传区域或将文件拖入，可多选。上传时显示：
- 进度条（百分比）
- 上传速度（KB/s）
- 预计剩余时间

上传完成后显示文件卡片，列出每个文件的行列数、列名及缺失情况。列名标签颜色区分：蓝色 = 数值型，灰色 = 文本型。

---

### 3.2 字段映射（可选，多文件时常用）

**适用场景**：两个文件都有"企业标识符"，但一个叫 `股票代码`，另一个叫 `证券代码`，需要统一成同一个名称才能合并。

操作：展开"字段映射"区块，找到需要重命名的列，输入新名称，回车确认。

---

### 3.3 合并配置（仅多文件时显示）

**合并方式**

| 方式 | 说明 | Stata 对应 |
|------|------|------------|
| 取交集（inner） | 只保留所有文件中都存在的记录 | `merge ..., keep(match)` |
| 左连接（left） | 以第一个文件为基准，匹配其他文件 | `merge ..., keep(master match)` |
| 取并集（outer） | 保留所有记录，未匹配处填缺失值 | `merge ..., keep(all)` |
| 纵向堆叠（concat） | 将文件行方向拼接（列名需一致） | `append using` |

**合并键**：选择用于匹配的列（如 `firm_id` + `year`）。选择后点击"检查合并类型"，系统自动判断：
- `1:1`：每个键值唯一，安全合并
- `1:N` 或 `N:1`：一对多关系，通常无问题
- `N:N`：多对多，会产生笛卡尔积，**建议检查数据后再合并**

---

### 3.4 清洗配置

#### 缺失值处理

| 选项 | 说明 |
|------|------|
| 删除行 | 删除含缺失值的行（默认） |
| 均值填充 | 用该列均值填充缺失值 |
| 中位数填充 | 用该列中位数填充 |
| 前向填充 | 用上一行的值填充（面板数据常用） |
| 填 0 | 将缺失值替换为 0 |

#### 异常值处理

| 选项 | 说明 | 阈值含义 |
|------|------|----------|
| 不处理 | 保留所有数据（默认） | — |
| Z-score | 删除标准化后绝对值超过阈值的行 | σ 倍数，默认 3 |
| IQR 法 | 删除超出 [Q1 − k×IQR, Q3 + k×IQR] 范围的行 | IQR 倍数，默认 1.5 |

#### 其他操作

- **强制文本型**：防止股票代码等带前导零的字段被识别为数字（如 `000001` → 变成 `1`）。选中相应列即可。
- **对数变换**：选择列后，自动生成 `ln_xxx` 新列（`log(原列)`），原列保留。
- **删除列**：选择不需要的列，执行清洗时一并删除。

#### 执行清洗

配置完成后点击**执行清洗**，完成后显示：
- 处理前后行数对比
- 缺失值处理数量
- 移除异常值行数
- 清洗后数据预览（前 10 行）

## 4. 第二层：实证分析

清洗完成后，页面自动显示第二层。

### 4.1 分析方法

可多选，同时执行多种分析：

| 分析方法 | 说明 |
|----------|------|
| 描述性统计 | Obs / 均值 / 标准差 / 最小值 / 中位数 / 最大值 / 偏度 / 峰度 |
| 相关系数矩阵 | Pearson 相关，含显著性标注（\*\*\* / \*\* / \*） |
| OLS 回归 | 普通最小二乘，支持多种标准误 |
| 固定效应（FE） | 控制个体固定效应，含 Hausman 检验 |
| 随机效应（RE） | GLS 估计，含 Hausman 检验 |

---

### 4.2 变量配置

**描述性统计 / 相关矩阵**：选择要纳入统计的变量（留空则使用所有数值列）。

**回归分析（OLS / FE / RE）** 共用同一套变量配置：

| 字段 | 说明 |
|------|------|
| 被解释变量 | Y 变量，只能选一个 |
| 核心解释变量 | 主要关注的 X 变量 |
| 控制变量 | 在表格中显示但优先级较低的变量 |
| 标准误类型 | 普通 / 稳健（HC1）/ 聚类 |
| 聚类变量 | 聚类 SE 时使用，只能选个体变量或时间变量 |

**面板分析额外字段**：

| 字段 | 说明 |
|------|------|
| 个体变量 | 标识企业/个人的列（如 `firm_id`、`stkcd`） |
| 时间变量 | 年份或期数列（如 `year`）；支持日期字符串，自动提取年份 |
| 时间固定效应 | 勾选后添加年份固定效应（双向 FE），仅对固定效应模型生效 |

> **注意**：变量旁边的徽章显示数据类型（float / int / 文本 / 日期），帮助判断变量是否适合纳入回归。文本型变量（object 类型）会**自动虚拟化**，等价于 Stata 的 `xi:` 前缀。

---

### 4.3 AI 解读（可选）

勾选"AI 解读"后，分析完成会自动调用 DeepSeek 大模型，从计量经济学视角解读系数含义和显著性。可输入自定义问题，引导解读方向（如"重点解释交互项的含义"）。

---

### 4.4 查看结果

#### 描述性统计表

| 列 | 含义 |
|----|------|
| Obs | 非缺失观测数 |
| Mean | 均值 |
| SD | 样本标准差（ddof=1，与 Stata 一致） |
| Min / Max | 最小值 / 最大值 |
| Median | 中位数 |

#### 相关系数矩阵

- 对角线为 1
- 括号外为相关系数，括号内为 p 值
- 显著性标注：`***` p<0.01，`**` p<0.05，`*` p<0.1

#### 回归结果表

- 系数下方括号内默认显示 **t 值**，可切换为**标准误**
- `***` / `**` / `*` 标注显著性
- 底部展示 N、R²（OLS）或组内 R²（面板）、F 统计量
- 如存在完全共线性，被自动移除的变量会在表底注明

#### 并列对比表

同时选择 **2 个及以上回归模型**（如 OLS + FE + RE）时，自动渲染 esttab 风格并列回归表，方便直接写入论文。

#### Hausman 检验

同时运行固定效应和随机效应时，自动执行 Hausman 检验并给出结论：
- p < 0.05：拒绝 RE，建议使用 FE
- p ≥ 0.05：不拒绝 RE，两者均可

## 5. 结果导出

点击分析完成后页面底部的导出按钮：

| 导出格式 | 内容 |
|----------|------|
| Excel (.xlsx) | 各分析结果分 Sheet，学术规范格式；同时运行多个回归时额外生成"回归对比" Sheet |
| Stata .do 文件 | 完整的清洗 + 分析代码，可在 Stata 中直接运行复现结果 |
| 纯文本 (.txt) | 结果文本，便于粘贴 |

## 6. 与 Stata 的对应关系

| 平台功能 | Stata 命令 |
|----------|------------|
| 描述性统计 | `summarize varlist, detail` |
| 相关系数矩阵 | `pwcorr varlist, sig star(0.1)` |
| OLS（稳健 SE） | `reg y x1 x2, robust` |
| OLS（聚类 SE） | `reg y x1 x2, cluster(firm_id)` |
| 固定效应 | `xtset firm_id year` + `xtreg y x1 x2, fe` |
| 双向固定效应 | `xtreg y x1 x2 i.year, fe` |
| 随机效应 | `xtreg y x1 x2, re` |
| Hausman 检验 | `hausman fe re` |
| 类别变量虚拟化 | `xi: reg y i.ind x1 x2` |
| 对数变换 | `gen ln_x = log(x)` |

## 7. 常见问题

**Q：上传 .dta 文件失败？**  
确认文件是 Stata 12 及以上格式（.dta 格式版本 115 或 117）。旧版 Stata 生成的 .dta 文件可先在 Stata 中 `saveold` 转换，或另存为 .csv。

**Q：股票代码合并后前导零丢失（`000001` 变成 `1`）？**  
在清洗配置的"强制文本型"中选中该列，执行清洗时会强制保留文本格式。

**Q：合并检查显示 N:N，怎么办？**  
N:N 会产生笛卡尔积（行数爆炸）。通常说明合并键不唯一，需要在原始数据中确认是否有重复记录，或考虑先聚合数据再合并。

**Q：固定效应结果没有常数项，正常吗？**  
正常。FE 模型通过个体去均值消除常数，Stata `xtreg, fe` 同样不报告截距。

**Q：运行面板回归时提示"某变量被移除"？**  
文本型列（如 `ind` 行业代码）虚拟化后，若与个体固定效应完全共线（如每个企业只属于一个行业），会被自动移除，与 Stata `absorb` 行为一致。可在结果表底部查看被移除的变量名。

**Q：时间变量是日期字符串（如 `2020-01-01`），能用吗？**  
可以。平台会自动提取年份（2020），以整数形式进入模型。

**Q：分析结果出现"—"（横杠）？**  
该单元格对应的值为缺失（如某列全为缺失导致统计量无法计算），属于正常显示。

**Q：页面报错"Application error"？**  
通常是浏览器缓存了旧版页面。按 **Ctrl + Shift + R**（Mac：Cmd + Shift + R）强制刷新即可。

**Q：清洗后重新分析，需要重新上传吗？**  
不需要。只需重新配置分析方法和变量，点击"开始分析"即可，清洗结果已缓存在服务器。但如果**刷新页面**或**重新上传文件**，缓存会清空，需重新执行清洗。
