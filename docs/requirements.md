# 论文实证分析平台 - 功能需求文档

## 一、项目背景

当前平台已实现基础的实证分析功能，但缺乏数据清洗和高级计量方法支持。本文档旨在梳理功能需求，为后续开发提供清晰的技术和业务指引。

### 3.2 文件合并

#### 3.2.1 功能描述
将多个上传的文件按照指定的键进行横向合并（增加变量）或纵向合并（增加观测值）。

#### 3.2.2 具体需求

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 横向合并 | P0 | 基于关键变量（如股票代码、年份）合并，增加变量数 |
| 纵向合并 | P1 | 将相同结构的文件追加观测值 |
| 合并预览 | P0 | 合并前显示合并后的数据结构预览 |
| 冲突检测 | P1 | 检测变量名冲突、重复键值等问题 |
| 合并日志 | P2 | 记录合并过程中的操作和警告信息 |

#### 3.2.3 合并规则
```
横向合并：
文件A (id, year, var1, var2) 
    +
文件B (id, year, var3, var4)
    →
结果 (id, year, var1, var2, var3, var4)
    键：id + year

纵向合并：
文件A (id, var1, var2)
    +
文件B (id, var1, var2)
    →
结果 (id, var1, var2) - 观测值相加
```

### 3.4 异常值处理

#### 3.4.1 功能描述

#### 3.4.2 具体需求

|--------|--------|------|

#### 3.4.3 Winsorize说明
```
原始数据：[1, 2, 3, 4, 5, 100, 200]
Winsorize (1%, 99%) 后：
- 1%分位数 = 1.5
- 99%分位数 = 180
结果：[1.5, 2, 3, 4, 5, 100, 180]
```

### 4.6 中介变量分析

#### 4.6.1 功能描述
检验变量的中介效应，分析变量间的作用机制。

#### 4.6.2 具体需求

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 中介效应检验 | P0 | 三步法检验中介效应 |
| Sobel检验 | P0 | Sobel中介效应检验 |
| Bootstrap检验 | P1 | Bootstrap置信区间法 |
| 中介效应分解 | P0 | 区分直接效应和间接效应 |
| 中介效应可视化 | P1 | 路径图展示中介机制 |

#### 4.6.3 中介效应模型（三步法）
```
步骤1：Y = c*X + ε         (总效应)
步骤2：M = a*X + ε         (X对中介变量的影响)
步骤3：Y = c'*X + b*M + ε  (直接效应 + 间接效应)

中介效应 = a * b
间接效应占比 = (a*b) / c
```

#### 4.6.4 输出格式
```
中介效应分析结果
因变量: roa          中介变量: innovation (创新投入)
============================================================

【第一步】总效应检验
roa = c*size + ε
c = 0.045*** (0.008), p<0.001

【第二步】中介变量检验
innovation = a*size + ε
a = 0.023*** (0.005), p<0.001

【第三步】直接效应+间接效应
roa = c'*size + b*innovation + ε
c' = 0.032*** (0.007), b = 0.568*** (0.045)

============================================================
中介效应检验
------------------------------------------------------------
间接效应 (a×b) = 0.023 × 0.568 = 0.013
Sobel Z值 = 4.52, p<0.001
Bootstrap 95%CI = [0.008, 0.019]
============================================================
结论：innovation在size对roa的影响中起到部分中介作用
      中介效应占总效应的比例为28.9%
```

### 4.7 DID分析（双重差分）

#### 4.7.1 功能描述
实现双重差分法，评估政策或事件的处理效应。

#### 4.7.2 具体需求

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| DID模型估计 | P0 | 标准DID回归：Y = α + β*Treat*Post + controls |
| 平行趋势检验 | P0 | 事件研究法(event study)检验平行趋势假设 |
| 安慰剂检验 | P1 | 随机政策时间或随机对照组 |
| PSM-DID | P2 | 倾向得分匹配后再做DID |
| 异质性分析 | P1 | 按地区、行业等分组进行异质性DID |
| 动态效应图 | P1 | 绘制政策前后的动态处理效应图 |

#### 4.7.3 DID模型设定
```
标准DID：
Y_it = α + β*(Treat_i × Post_t) + γ*X_it + μ_i + λ_t + ε_it

其中：
- Treat_i: 处理组虚拟变量（1=实验组，0=对照组）
- Post_t: 政策实施后虚拟变量（1=政策后，0=政策前）
- β: DID估计量（核心关心的处理效应）
- X_it: 控制变量
- μ_i: 个体固定效应
- λ_t: 时间固定效应
```

#### 4.7.4 输出格式
```
DID回归分析结果
因变量: roa          处理组: 政策试点企业    对照组: 非试点企业
政策时间: 2015年
============================================================
                (1)DID        (2)倾向得分匹配+DID
------------------------------------------------------------
Treat×Post      0.025***       0.031***
              (0.006)         (0.007)

size           -0.001          0.002
              (0.003)         (0.004)

lev             0.012          0.008
              (0.015)         (0.018)

Constant        0.056***       0.048***
              (0.008)         (0.009)

------------------------------------------------------------
Observations     5000           3200
R-squared        0.245         0.198
============================================================
注：* p<0.1, ** p<0.05, *** p<0.01；控制了个体和时间固定效应
```

#### 4.7.5 平行趋势检验输出

## 五、非功能性需求

### 5.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 文件上传 | 单文件≤50MB | 超过需提示用户拆分 |
| 数据处理速度 | 10万行数据≤5秒 | 含缺失值和异常值处理 |
| 回归分析速度 | 1万观测值≤3秒 | 含固定效应估计 |
| 页面响应时间 | ≤2秒 | 用户操作的响应 |

### 5.2 兼容性需求

| 项目 | 要求 |
|------|------|
| 浏览器支持 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 数据格式 | CSV, Excel (.xlsx/.xls), Stata (.dta) |
| 响应式设计 | 支持PC端和平板端 |

### 5.3 数据安全

| 项目 | 要求 |
|------|------|
| 数据存储 | 本地浏览器，不上传到服务器 |
| 临时文件 | 关闭页面后自动清除 |
| 敏感数据 | 不记录、不追踪用户数据 |

6. 数据导出功能

### 第三阶段
1. DID分析
2. 平行趋势检验
3. 安慰剂检验

6. 可视化功能

## 七、技术选型建议

### 7.1 前端框架
- **当前方案**：Next.js + React（已采用）
- **优势**：SSR支持、API路由、生态成熟

### 7.2 数据处理
- **浏览器端数据处理**：JavaScript (DataFrame.js, Danfo.js)
- **优点**：纯前端处理，保护数据隐私

### 7.3 AI集成
- **当前方案**：通义千问API（已采用）
- **扩展**：考虑接入更多模型支持复杂计量分析

### 7.4 数据存储
- **浏览器存储**：IndexedDB
- **用途**：存储清洗后的面板数据、管理用户工作区

### 7.5 可视化
- **图表库**：Chart.js / ECharts
- **用途**：相关性热力图、回归系数图、DID动态效应图

## 八、用户操作流程

```
1. 上传数据
   └─→ 上传多个原始数据文件
   └─→ 系统显示文件列表和预览

2. 数据清洗
   └─→ 合并多个文件（如需要）
   └─→ 设置面板数据结构
   └─→ 处理缺失值
   └─→ 处理异常值
   └─→ 变量管理
   └─→ 导出/保存清洗后数据

3. 实证分析
   └─→ 选择分析类型
   ├─→ 描述性统计
   ├─→ 相关性分析
   ├─→ 基础回归
   ├─→ 调节效应
   ├─→ 中介效应
   └─→ DID分析

4. 结果输出
   └─→ 查看分析结果
   └─→ 复制/下载结果表格
   └─→ 导出完整分析报告
```
